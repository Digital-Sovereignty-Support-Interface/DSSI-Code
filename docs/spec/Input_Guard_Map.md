# 入力ガード定義マップ (Input Guard Logic Map)

| 対象フィールド (HTMLタグ/属性) | リスクレベル | 推奨アクション (Guide) | 表示すべき技術チップス (Tech Tip) |
| :--- | :--- | :--- | :--- |
| `input[type="password"]` | **高 (High)** | パスワードマネージャー使用 | 「キーロガー対策のため、手入力よりペースト/マネージャー入力を推奨します。」 |
| `input[name="cc-number"]` | **高 (High)** | HTTPS確認 + マネージャー | 「非HTTPS環境でのカード入力は通信傍受のリスクがあります。鍵マークを確認してください。」 |
| `input[type="email"]` | 中 (Mid) | フィッシング確認 | 「ここは連絡先入力欄です。ドメイン名が正規のものか確認しましたか？」 |
| `input[type="search"]` | 低 (Low) | 制限なし | (表示なし) |
| `button[type="submit"]` | **特大 (Critical)** | ワンクッション停止 | 「これは契約の確定ボタンです。規約の『高リスク条項』を確認しましたか？」 |
